# 福利厚生解読スキル

「トータル報酬」の資料はマーケティングだ。このスキルは、一度痛い目を見た友人のようにプラン条項を読む：各福利厚生が実際いくらの価値か、どの約束に逃げ道が仕込まれているか、サインする前に何を書面で取るべきか。

## このスキルが産出するもの

- 計算可能なものは実質年額を付した、項目別の福利厚生解読
- ランク付けされた危険信号：ベスティングクリフ、返還条項、「裁量」だらけの文言、補償のギャップ
- 401k/年金マッチングの計算とエクイティの計算、算式は見えるところに
- サイン前に人事へ聞く質問——そしてどの回答を書面で取るか

## 必要な入力

提供されていない場合のみ求める：

- **福利厚生の書類**——オファーレター、福利厚生サマリー、エクイティ付与条件、プラン抜粋。提供された分を解読し、まだ足りないもの（エクイティプラン、保険給付サマリー、ボーナスプラン条件）を列挙する。
- **基本給とエクイティ付与の詳細**（本文にない場合）——計算に必要。
- **本人の状況**——扶養家族/健康上のニーズ、現実的にどれくらい在籍するつもりか。

## フレームワーク：深刻度スケール

- 🔴 **実際にお金を失わせ得る**——予想在籍期間に対するベスティングクリフ、返還条項（サインボーナス、転居費、学費、「正当事由」トリガーでは既得エクイティまで）、「支払日に在籍」の場合のみ支払われるボーナス、退職後の短すぎる行使期間、良い見出しの裏の高い免責額、ベスティングによるマッチング没収。
- 🟡 **異例——サイン前に明確化**——「裁量」ボーナスの文言（率直に解読：目標であって約束ではない）、無制限休暇（経済学を解読：退職時の未消化分の支払いがない）、「会社の裁量で」変更可能な福利厚生、待機期間。
- 🟢 **標準**——通常の加入期間、標準的なベスティング形状、典型的なプラン定型文。問題ないと分かるようラベルを付ける。

計算は常に見せる：
1. **マッチングの計算**——例「最初の6%の50%」= 本人の給与で年X。マッチング自体のベスティングと、N年目に辞めた場合の没収額を注記。
2. **エクイティの計算**——付与額 ÷ ベスティング年数 = 表明バリュエーションでの年価値。クリフのシナリオ付き（「11か月目に退職 = 0」）。バリュエーション依存の数字は `[要確認]`。
3. **保険の実質補償の読み**——保険料負担、免責額、自己負担上限：パンフレットの一行ではなく、最悪の年を金額で。
4. **休暇の経済学**——無制限 vs 積立：退職時の支払い差を金額で。

## 出力形式

### 福利厚生解読：[会社 / オファー]

**1. 評決**——このパッケージが年間で実際いくらの価値か（レンジ、前提を明示）、そしてサイン前に解決すべき二つのこと。

**2. 項目別解読**

| 福利厚生 | 書類の記載 | 実際の価値 / 実際の意味 | 深刻度 |
|---|---|---|---|

**3. 🚩 危険信号、ランク順**——引用した文言、牙をむくシナリオ、金額のコスト。

**4. 計算セクション**——マッチング、エクイティ、保険の最悪ケース、休暇——算式を見せる。

**5. サイン前の人事への質問**——金額の大きい順に4〜7個。どの回答を書面で取るか印を付ける（ボーナス条件、エクイティプラン文書、返還トリガー）。

**6. 交渉可能なもの**——通常はプラン自体より一時金項目（サインボーナス、エクイティ、入社日、転居費）。

成果物の末尾に、一字一句そのまま付す：*「これは平易な言葉での読解であり、法的・財務的助言ではありません——法律は法域により異なります。重要事項は資格を持つ専門家に確認してください。」*

## 品質チェック

- [ ] すべての評価額が前提付きの算式で示され、断言されていない
- [ ] 「裁量」と「支払日に在籍」の文言が引用され、率直に解読されている
- [ ] クリフ/返還の危険信号が本人の予想在籍期間に結びついている
- [ ] 欠けている書類と検証不能な数字が `[要確認]` として列挙されている
- [ ] 本当に標準的な条件は 🟢 と表示——すべてが罠ではない
- [ ] 免責文が一字一句そのまま成果物に現れる

## アンチパターン

- [ ] 書類にない福利厚生や条件を創作しない
- [ ] バランスを装って危険信号を和らげない——「裁量」はボーナスの支払い義務がないという意味。そう言う
- [ ] 法域依存のルール（休暇の支払い、返還条項）を普遍として提示しない
- [ ] バリュエーション前提の印なしにエクイティを額面で受け取らない
- [ ] 「トータル報酬」の見出しを立たせたままにしない——プラン条項から再構築する

## 基盤

オファーレビュー実務——トータル報酬の再構築、プラン文言の解読、サイン前質問リスト。
