# 支払拒否解読スキル

拒否通知は会話を終わらせるために書かれる。不服申立てが存在するのは、会話がしばしば終わるべきでないからだ。相当な割合の拒否——特にコーディングエラー、「医療上不要」、書類不備——は、誰かが*引用された理由*に怒りではなく証拠で答えたときに覆る。このスキルは拒否が実際に何を主張しているかを解読し、各主張にそれへ答える証拠を対応させ、エスカレートする人間が書いたと読める申立書を書く。

## このスキルが産出するもの

- 拒否の解読：引用された理由の平易な言葉版と、その理由タイプが勝算に意味すること
- 拒否タイプに対応した証拠チェックリスト——書く前に集めるもの
- 申立書——一点ずつ反論、書類を逐一引用、期限を意識
- エスカレーションの梯子——内部申立ての階層、外部/独立審査、監督当局への苦情——順番に

## 必要な入力

提供されていない場合のみ求める：

- **拒否通知のテキスト**——引用された理由コード/文言と、明記された申立て期限。まだ通知がない？最初の一手は、具体的な証券上の根拠を付した書面での拒否を求めること。
- **請求の経緯**——何を、いつ請求したか、損害または治療、事前承認や査定人とのやり取り。
- **証券の文言**（あれば）——申立ての最強の文は、証券で拒否を打つもの。
- **既に送ったもの**——そして保険会社が「受け取っていない」と主張するもの（お決まりの一幕）。

## フレームワーク：深刻度スケール

拒否の引用理由を読み、分類する：

- 🔴 **通常は答えられる——証拠を付けて申し立てる**——「医療上不要」（主治医の書簡+記録+ガイドラインで答える）、コーディング/請求エラー（医療機関の修正コードで答える）、「書類未着」（配達証明付きの再送で答える）、証拠なしの「既往症」「既存損傷」の断定（日付入り記録で答える）、低すぎる評価額（独立見積もりで答える——これは拒否ではなく争議であり、手紙で再定義すべき）。
- 🟡 **より困難だが争える**——「実験的/研究段階」の治療（ガイドラインと査読文献の裏付けが必要）、正当事由のある遅延申請、証券文言が曖昧な除外の解釈（曖昧さは一般に起草側に不利に読まれる——法域依存のフラグ付き、争う価値あり）。
- 🟢 **おそらく妥当な拒否**——明確に除外された危険/治療、損害日に失効していた証券、保険期間外の請求。正直にそう言う——見込みのない申立ては、機能し得る選択肢（交渉、分割払い）の期限を燃やす。

申立ての構造は常に同じ：**拒否の原文を復唱 → その理由に添付証拠で具体的に答える → 補償を支持する証券文言を引用 → 具体的な要求を述べる → 期限内であることと、無回答の場合の次の一手を名指しする。**通知が引いていない理由には決して触れない——新しい拒否理由を教えることになる。

## 出力形式

### 拒否解読と申立て：[請求番号、保険会社]

**1. 解読**——引用理由の平易な言葉版、タイプ、正直な勝算の枠組み（答えられる/争える/おそらく妥当）。

**2. 証拠チェックリスト**

| 拒否の主張 | それに答える証拠 | 入手先 | 状態 |
|---|---|---|---|

**3. 申立書**——送付可能な全文：請求情報のヘッダー · 「貴社[日付]付の通知には『[引用]』とあります」 · 添付番号付きの一点ずつの反論 · 証券文言の引用 · 具体的要求 · 追跡可能な送付手段の明記。

**4. エスカレーションの梯子**——内部申立ての階層と期限 → 外部/独立審査の権利（プランと法域に依存——拒否通知自身の権利告知セクションを読む）→ 監督当局への苦情 → 督促状/弁護士の閾値。

**5. 期限ボックス**——通知と証券にある重要な日付すべて。

成果物の末尾に、一字一句そのまま付す：*「これは平易な言葉での読解であり、法的・財務的助言ではありません——法律は法域により異なります。重要事項は資格を持つ専門家に確認してください。」*

## 品質チェック

- [ ] 申立てが拒否の原文を引用して復唱し、それに答えている
- [ ] 申立て内のすべての主張が添付書類を指している
- [ ] 勝算の枠組みが正直——見込みのない拒否は代替経路とともに名指しされている
- [ ] エスカレーションの梯子は、どの段が法域/プラン依存かを注記している
- [ ] すべての期限が一つのボックスに抽出され、送付日の推奨が期限内にある
- [ ] 免責文が一字一句そのまま成果物に現れる

## アンチパターン

- [ ] 怒りの手紙を書かない——査定人は怒りを山の底へ回す。証拠がファイルを動かす
- [ ] 通知が引いていない理由を争わない——主張されたことだけに答える
- [ ] 覆る確率をパーセンテージで約束しない——統計を捏造せず、理由タイプのパターンとして示す
- [ ] 完璧な証拠集めで期限を逃さない——期限内に十分なものを出し、後から補強する
- [ ] 評価額争議の再定義を省かない——「拒否」と「低額査定」は別のプレイブック

## 基盤

契約者側の不服申立て実務——拒否理由のトリアージ、証拠マッチング、一点ずつの申立て起草、エスカレーション設計。
