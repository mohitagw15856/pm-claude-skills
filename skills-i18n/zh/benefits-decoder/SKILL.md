# 福利解读技能

"总薪酬"演示文稿是营销材料。这个技能像一位吃过亏的朋友那样读计划条款：每项福利真正值多少钱、哪些承诺留了后门、签字前该把什么落到书面上。

## 这个技能产出什么

- 逐项福利解读，可计算处给出真实年化价值
- 分级警示信号：归属悬崖、追回条款、满篇"酌情"、保障缺口
- 401k/养老金匹配的账和股权的账，算式摆在明面
- 签字前要问HR的问题——以及哪些答案要拿到书面确认

## 所需输入

仅在未提供时索取：

- **福利文件**——offer信、福利摘要、股权授予条款、计划节选。解读已提供的；列出还缺的（股权计划、保险福利摘要、奖金计划条款）。
- **基本薪资和股权授予明细**（若文本中没有）——算账需要。
- **他们的处境**——家属/健康需求、实际预计待多久。

## 框架：严重程度分级

- 🔴 **会让你损失真金白银**——归属悬崖对照预期任职时间、追回条款（签字费、搬迁费、学费，甚至"因故"触发时已归属的股权）、仅在"支付日仍在职"才发放的奖金、离职后过短的行权窗口、好听数字背后的高免赔额、匹配因归属而被没收。
- 🟡 **不寻常——签字前澄清**——"酌情"奖金措辞（直白解读：是目标，不是承诺）、无限假期（解读其经济学：离职时没有未休假折现）、"公司有权酌情调整"的福利、等待期。
- 🟢 **标准**——正常的登记窗口、标准的归属曲线、典型的计划模板；标注出来，让读者知道哪些没问题。

账目永远摆出来：
1. **匹配的账**——如"前6%的50%"= 按其薪资每年X；注明匹配本身的归属，以及第N年离开会没收多少。
2. **股权的账**——授予额 ÷ 归属年数 = 按声明估值的年价值，附悬崖情景（"第11个月离开 = 0"）；依赖估值的数字标注 `[待确认]`。
3. **保险实际保障解读**——保费分摊、免赔额、自付上限：最坏一年的真实金额，而非宣传册上的那行。
4. **假期经济学**——无限 vs 累积：离职折现差额，用钱表示。

## 输出格式

### 福利解读：[公司 / offer]

**1. 结论**——这个包每年真正值多少（区间，标明假设），以及签字前必须解决的两件事。

**2. 逐项福利解读**

| 福利 | 文件怎么写 | 真正值多少 / 真正意味着什么 | 严重程度 |
|---|---|---|---|

**3. 🚩 警示信号，分级排序**——引用的措辞、咬人的情景、金额代价。

**4. 算账部分**——匹配、股权、保险最坏情况、假期——算式摆出来。

**5. 签字前问HR的问题**——4–7个，按涉及金额排序；标注哪些答案要书面确认（奖金条款、股权计划文件、追回触发条件）。

**6. 什么可以谈**——通常是一次性项目（签字费、股权、入职日期、搬迁）而非计划本身。

在产出物结尾原文附上：*"这是一份大白话解读，不构成法律/财务建议——法律因司法辖区而异；任何关键事项请向合格专业人士确认。"*

## 质量检查

- [ ] 每项估值都以带假设的算式呈现，而非断言
- [ ] "酌情"与"支付日在职"措辞被引用并直白解读
- [ ] 悬崖/追回警示与用户声明的预期任职时间挂钩
- [ ] 缺失文件与无法核实的数字列为 `[待确认]`
- [ ] 真正标准的条款标为 🟢——不是所有东西都是陷阱
- [ ] 免责声明逐字出现在产出物中

## 反模式

- [ ] 不要虚构文件中不存在的福利或条款
- [ ] 不要为显得客观而软化警示信号——"酌情"意味着不欠任何奖金；直说
- [ ] 不要把依赖司法辖区的规则（假期折现、追回）说成普适
- [ ] 不要不加估值假设标注就照单全收股权数字
- [ ] 不要让"总薪酬"标题立在那里——从计划条款重新算一遍

## 依据

Offer审查实践——总薪酬重构、计划条款解读、签字前问题清单。
